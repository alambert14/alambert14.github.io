OI

The OI file is where all of the button commands are set. The OI is a class that can be instantiated, but it also needs to be initialized by running the initialize() method. When you create your robot project, an OI class will be created for you, with an initialize() method. First, you start by declaring all of the JoystickButton variables that you need. In the initialize method, you will start by setting these variables equal to a button created in RobotMap. Then, you will call a method, such a whileHeld() or whenPressed(), depending on what kind of function the button handles. For example, whileHeld() might be used for a ball intake mechanism, if you want the motor to spin while holding down a button. whenPressed() works best with a toggle, or single action, such as firing a piston. Inside of this method, there is a parameter required: a Command class. This command is what you wish to execute when the button is fired. You simply do this by creating a new Command object inside of the parentheses. Here is an example of a basic OI class:

public class OI  {
private static Button aButton;
public static void initialize() {
    aButton = RobotMap.A_BUTTON;
    aButton.whileHeld(new MyCommand());
}
}
