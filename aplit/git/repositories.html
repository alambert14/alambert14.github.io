Repositories

The concept of “repository” is super important to understand in terms of using Git and other version control in general. Basically it is where all of the code is stored together. Typically, it will have a remote location on the Internet, and then local copies on whichever computers are collaborating.

You can create a repository with any local folder, but but it must be linked to an online repository (typically called an origin). This will allow your computer to upload and download code from the cloud.

When you create a robot project in Eclipse, it is pretty simple to turn it into a repository, but you can also link an existing online repository to your robot project. We will talk more about how to create an online repository later on.
