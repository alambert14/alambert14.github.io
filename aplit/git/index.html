Git

The thing about programming a robot is that it's hard to do on your own. There are so many moving parts and you only have six weeks to go! This is why collaboration is key, and Git is a program which provides the means to do so. A good way to think of Git is as a tool that allows you to download code from the cloud, and so can others. This way, multiple collaborators can download the same code, make changes, and send the code back to the cloud for others to see. However, you might see this can get complicated: if two people make different changes to the same code, which one is saved?

You can download git here. It is important that it is installed on each computer that will be collaborating. When Eclipse is downloaded, Git should be integrated into the IDE, making it much easier to use and understand.

If this guide ends up being incomprehensible (as it very well may be), then I would also recommend the Codecademy tutorials for Git. These are all based in a Unix command line, but it is definitely helpful in terms of understanding the concepts needed to use Git effectively. The truth about Git is that it really just takes practice, practice, and more practice. However, it is a widely used tool that will be very helpful for future internships, jobs, or personal projects!
